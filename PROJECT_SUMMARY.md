## 🎯 核心价值

### 与现有客户端的区别

| 特性 | NekoBox/Hiddify | OneProxy |
|------|-----------------|----------|
| **设计目标** | 单一出口，自动切换 | **多端口并行，固定绑定** |
| **节点使用** | 同时只能用一个 | **所有节点同时可用** |
| **切换方式** | GUI 点击 | **直接换端口号** |
| **应用分流** | 复杂规则 | **手动指定端口** |
| **健康监控** | 自动切换 | **显示状态，不切换** |
| **故障恢复** | 被动等待 | **主动DNS刷新** |

### 真实使用场景

1. **开发测试** - 同时测试多个节点延迟
2. **多应用分流** - 不同应用使用不同节点
3. **快速切换** - 端口号一改就换节点
4. **团队共享** - 内网暴露，隐藏密码
5. **自动恢复** - IP 变更后自动刷新

## 🔧 技术架构

### 核心设计

```
用户配置 (config.json)
    ↓
配置加载与验证
    ↓
sing-box配置生成 (一对一绑定)
    ↓
sing-box进程启动
    ↓
健康检查模块启动
    ↓
并发检查所有节点
    ↓
延迟测量 + 状态记录
    ↓
连续失败3次？
    ↓ 是
DNS刷新 + sing-box重启
    ↓
自动恢复
```

### sing-box 配置

**5个独立inbound**:
```json
{
  "inbounds": [
    {"type": "socks", "listen": "127.0.0.1", "listen_port": 10801, "tag": "in-10801"},
    {"type": "socks", "listen": "127.0.0.1", "listen_port": 10802, "tag": "in-10802"},
    {"type": "socks", "listen": "127.0.0.1", "listen_port": 10803, "tag": "in-10803"},
    {"type": "socks", "listen": "127.0.0.1", "listen_port": 10804, "tag": "in-10804"},
    {"type": "socks", "listen": "127.0.0.1", "listen_port": 10805, "tag": "in-10805"}
  ]
}
```

**一对一固定绑定**:
```json
{
  "route": {
    "rules": [
      {"inbound": ["in-10801"], "outbound": "out-JMS-Server1"},
      {"inbound": ["in-10802"], "outbound": "out-JMS-Server2"},
      {"inbound": ["in-10803"], "outbound": "out-JMS-Server3"},
      {"inbound": ["in-10804"], "outbound": "out-JMS-Server4"},
      {"inbound": ["in-10805"], "outbound": "out-JMS-Server5"}
    ]
  }
}
```

## 🚀 快速开始

### 1. 下载 sing-box
```bash
./download-singbox.bat  # Windows
```

### 2. 配置代理
```bash
cp configs/config.example.json config.json
# 编辑 config.json，填入真实的代理信息
```

### 3. 运行
```bash
./oneproxy.exe
```

### 4. 使用
```bash
# 浏览器：SwitchyOmega → 127.0.0.1:10801
# 命令行：
curl -x socks5://127.0.0.1:10801 https://google.com
curl -x socks5://127.0.0.1:10802 https://google.com
```

## 📝 使用建议

### 推荐配置

```json
{
  "health_check": {
    "enabled": true,
    "interval_seconds": 60,
    "timeout_seconds": 5
  },
  "dns": {
    "flush_on_failure": true,
    "servers": [
      "https://1.1.1.1/dns-query",
      "https://8.8.8.8/dns-query"
    ]
  },
  "inbound": {
    "listen": "127.0.0.1",  // 仅本地，安全
    "proxy_type": "socks5"
  }
}
```

### 最佳实践

1. **使用域名** - JustMySocks域名自动更新
2. **启用健康检查** - 自动监控节点状态
3. **启用DNS刷新** - IP变更快速恢复
4. **合理分配端口** - 不同应用用不同端口
5. **定期查看日志** - 了解节点状态
